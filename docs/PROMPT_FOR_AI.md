## 🎯 기대 답변 (정답)

### 1. 시스템 아키텍처 3단계
```
Phase 1: 개별 크롤러 구축 (24개)
  - 18개 자사사이트 + 6개 플랫폼

Phase 2: 사업자 통합 모듈 (18개)
  - 각 사업자별로 자사 + 제휴 플랫폼 통합
  - 중복 제거

Phase 3: 전체 자동화 스케줄러
  - 18개 사업자 모듈을 시간별 자동 실행
```

### 2. 필수 9개 필드
```
carrier, plan_name, data, voice, sms,
price_regular, price_promo, network, network_type
```

### 3. Simple vs Advanced Mode
```
Simple Mode:
- YAML 설정만으로 작동
- UniversalCrawler 사용
- 코드 작성 불필요

Advanced Mode:
- Python 코드 작성 필요
- 복잡한 필터링/페이지네이션
- BaseCrawler 상속
```

### 4. 사업자 통합 모듈 역할
```
- 사업자의 자사사이트 크롤링
- 제휴된 플랫폼에서 해당 사업자 요금제 수집
- 다중 출처 데이터 통합 및 중복 제거
- 우선순위 기반 데이터 선택
- 통합 Excel 생성
```

### 5. 중복 제거 알고리즘
```
1. plan_name + data 기준으로 중복 체크
2. 동일 요금제 발견 시 우선순위 비교
3. 우선순위 높은 플랫폼 데이터 선택
   (자사사이트 > 폰비 > 모요 > ...)
```

### 6. Week 2 MVP
```
- 헬로모바일 자사사이트 크롤러
- 폰비 플랫폼 크롤러
- 2개 크롤러로 통합 테스트
- 표준 스키마 25개 필드 수집 검증
```
